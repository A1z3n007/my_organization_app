# Neon CRM

Neon CRM — это мини‑CRM на Flutter с тёмной неоновой темой и локальным хранилищем (SharedPreferences). Приложение помогает вести клиентов, сделки, задачи и заметки без подключения к серверу: вся информация сохраняется в памяти устройства, поэтому после повторного запуска вы сразу попадаете в рабочее пространство.

## Возможности

- **Регистрация и вход.** Простая аутентификация по email и паролю. После входа пользователь остаётся в системе, пока явно не нажмёт «Выйти».
- **Обзор (Dashboard).** Карточки с ключевыми метриками (заметки, задачи, клиенты, сделки), быстрые действия, список ближайших задач и «горящие» сделки.
- **Заметки.** Быстрые записи с заголовком и описанием. Список сортируется по дате создания.
- **Задачи.** Название, описание, дедлайн и приоритет; переключение статуса «в работе/готово». Список сортируется по ближайшему сроку.
- **Клиенты.** Имя, компания, контактные данные и этап взаимоотношений.
- **Сделки.** Несколько стадий (Новый → Контакт → Переговоры → Согласование → Успешно), сумма, дедлайн и следующий шаг. Любую сделку можно быстро перевести на другой этап.
- **Диалоги создания.** Все формы открываются во всплывающих окнах с валидацией вводимых данных и выбором даты.
- **Тёмная неоновая тема.** Общий стиль выдержан на базе Material 3; нижняя навигация и карточки адаптированы под тёмный фон.

## Технологии

| Компонент | Назначение |
|-----------|-----------|
| Flutter 3.10+ | UI, навигация между вкладками, Material 3 |
| Dart | Бизнес-логика и состояние приложения |
| `shared_preferences` | Локальное хранение данных пользователей, заметок, задач, клиентов и сделок |
| `ChangeNotifier` / `AnimatedBuilder` | Реактивное обновление экрана при изменении состояния (`AppState`) |

## Установка и запуск

1. Установите Flutter (версия ≥ 3.10) и необходимые SDK платформ.
2. Клонируйте проект:
   ```bash
   git clone https://github.com/<user>/my_organization_app.git
   cd my_organization_app
   ```
3. Загрузите зависимости:
   ```bash
   flutter pub get
   ```
4. Запустите приложение на эмуляторе или подключённом устройстве:
   ```bash
   flutter run
   ```

## Сценарий использования

1. **Регистрация или вход.** Заполните форму (имя, email, пароль ≥ 4 символов). После регистрации создаётся демонстрационный набор данных.
2. **Рабочий стол.** На вкладке «Обзор» доступны быстрые кнопки «Заметка», «Задача», «Сделка». Здесь же отображаются метрики и ближайшие дела.
3. **Ведение базы.**
   - **Заметки.** Используйте для фиксации мыслей и комментариев.
   - **Задачи.** Планируйте дедлайны и отмечайте выполнение.
   - **Клиенты.** Сохраняйте ключевую информацию и этап взаимодействия.
   - **Сделки.** Отслеживайте прогресс воронки и дату следующего шага.
4. **Выход.** Кнопка в правой части AppBar. Данные остаются в памяти устройства (`SharedPreferences`), повторный вход занимает секунды.

## Полезные команды

- `flutter analyze` — статический анализ кода.
- `flutter test` — запуск тестов (в проекте есть базовый виджет‑тест).
- `flutter run` — запуск на выбранном устройстве/эмуляторе.
- `flutter build apk` или `flutter build web` — сборка релизных артефактов.
